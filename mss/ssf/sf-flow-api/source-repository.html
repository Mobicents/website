sf-flow-api

概要

このプロジェクトは Subversion をソースコード管理に使用しています。Subversion の使用法については http://svnbook.red-bean.com/ を参照してください。

ウェブアクセス

以下はオンラインソースリポジトリへのリンクです。

http://172.28.43.220/svn/wallaby/ssf/trunk/sf-flow-api/

匿名アクセス

ソースはこのコマンドによって SVN から匿名でチェックアウトできます。

$ svn checkout svn://172.28.43.220/wallaby/ssf/trunk/sf-flow-api/ sf-flow-api

開発者アクセス

Subversion リポジトリにはだれでも SVN でアクセスできますが、 コミットは Subversion リポジトリから SVN でチェックアウトしなければなりません。

$ svn checkout svn://172.28.43.220/wallaby/ssf/trunk/sf-flow-api/ sf-flow-api

リポジトリに変更をコミットするには、, 次のコマンドを実行してください。(svn はパスワードの入力を求めます)

$ svn commit --username your-username -m "A message"

ファイアフォールの後ろからのアクセス

企業のファイアウォールに Subversion リポジトリへの HTTP アクセスをブロックされて困っているユーザは、開発者用の接続を試すことができます。

$ svn checkout svn://172.28.43.220/wallaby/ssf/trunk/sf-flow-api/ sf-flow-api

プロキシ経由のアクセス

適切に設定されれば、サブバージョンはプロキシを通り抜けることが出来ます。まず、"servers" 設定ファイルを編集し、使用するプロキシを設定してください。ファイルの場所はオペレーティングシステムに依存します。 Linux または Unix では "~/.subversion" に配置されます。Windowsでは "%APPDATA%\Subversion" に配置されます。(このディレクトリが隠しディレクトリであることに注意しましょう。"echo %APPDATA%"を試してください。)

このファイルには何をすればよいか説明するコメントが書いてあります。このファイルが無かったら、最新の Subversion クライアントを取得し、なにかコマンドを実行してください。それによって設定ディレクトリが作成され、テンプレートファイルが生成されるでしょう。

例 : 'servers' ファイルを次のように編集してください。

[global]
http-proxy-host = your.proxy.name
http-proxy-port = 3128
